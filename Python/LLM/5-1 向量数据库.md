# 向量数据库

## 基本概念
向量数据库是专门用于存储、索引和检索高维向量的数据库系统。
在LangChain中，向量数据库主要用于存储文档的嵌入向量，支持高效的相似性搜索。


## 分类

### 企业级云原生分布式（巨量/高可用）


- Milvus: 采用存算分离架构，支持超大规模数据（十亿级以上），具备极强的水平扩展能力和多租户安全控制。它是目前国内企业级私有化部署的首选。
- Pinecone: 全托管的 SaaS 服务，完全屏蔽了底层运维。适合追求极致开发效率、不希望投入资源维护服务器、且数据可以放在公有云的团队。


### 高性能原生向量搜索引擎（中大规模/极速）

- Qdrant: 使用 Rust 编写，内存效率极高，在高并发下延迟表现极佳。支持复杂的元数据过滤，适合对响应速度有严苛要求的实时生产环境。
- Weaviate: 基于 Go 开发，侧重于语义索引和多模态支持。它自带丰富的一体化模块（如自动文本转向量），更像是一个以向量为核心的“语义知识库”。

### 极简轻量级工具（实验/开发期）

- Chroma: 定位是“AI 开发者的数据库”，主打简单好用，几行代码即可完成配置。非常适合个人开发者做 Demo、原型验证或中小型本地 RAG 应用。

### 存量架构升级

- Elasticsearch: 通过引入 HNSW 插件支持向量检索,适合非结构化文本检索
- Pgvector: 适合关系型数据，追求数据一致性

## Embedding

### 第一梯队（工业级 RAG 首选）

- BGE-M3: 目前中文社区的“六边形战士”。支持 8k 超长上下文，且同时支持稠密检索、稀疏检索（类似关键词）和多向量重排
- GTE-Qwen / GTE-MLM: 阿里出品。目前的 GTE 系列（尤其是基于 Qwen 架构的新版）支持 8k 以上的超长上下文
- OpenAI text-embedding-3: 闭源商业标杆。不仅精度极高，还支持“向量缩减”技术（可以在不重训的情况下调整维度以节省成本），适合不追求私有化部署、预算充足的团队


### 第二梯队（中文语义调优）

- BGE-large-zh: 智源研究院在 M3 之前的巅峰之作，专注中文。虽然只支持 512 tokens，但在处理短文本相似度、语义检索上的纯精度表现依然非常能打


### 第三梯队（轻量级）

- M3E-base: 号称“小而美”。由于使用了大量中文社区语料（如知乎、新闻）训练，它对中文口语和百科类知识的理解非常出色，且计算开销远小于 BGE 系列 。
- text2vec-base-chinese: 早期的“国民模型”。基于 CoSENT 架构，代码极其精简，适合作为入门学习或极简单的句子对匹配工具 。
