# UV
uv 是由 Astral 团队使用 Rust 语言开发的一款超高速 Python 包管理器与环境管理器。
它旨在作为 pip、pip-tools、virtualenv、pyenv 以及 poetry 等工具的即插即用式跨平台替代品。
uv 不仅支持包的安装，还能一站式完成 Python 版本管理、虚拟环境创建、依赖解析以及项目配置管理。

## 安装部署

MacOS/Linux
```shell
curl -LsSf https://astral.sh/uv/install.sh | sh
```

Windows
```shell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

验证安装
```shell
uv --version
```
若提示`uv command not found`,重新加载环境变量或重启终端

## 基础环境管理

### 创建虚拟环境

在项目目录下运行以下命令，默认会创建一个名为 `.venv` 的环境
```shell
uv venv
```

如果需要指定 Python 版本（若本地未安装该版本，uv 会自动下载），可以使用

```shell
uv venv --python 3.11
```

### 激活虚拟环境

- Linux/macOS: `source .venv/bin/activate`
- Windows (PowerShell): `.\.venv\Scripts\activate`

## 包管理

`uv pip` 提供了与 pip 高度兼容的命令格式，但速度极快

```shell
uv pip install requests pandas
```

从 `requirements.txt` 安装
```shell
uv pip install -r requirements.txt
```

卸载依赖包
```shell
uv pip uninstall requests
```

升级依赖包

```shell
uv pip install --upgrade requests
```

导出当前环境依赖
```shell
uv pip freeze > requirements.txt
```

## 严格同步环境
在使用传统的 pip install -r requirements.txt 时，如果环境中存在多余的包，
它们不会被自动删除，这容易导致线上与线下环境不一致。uv 提供了更严格的同步命令来解决这个问题。

严格同步环境：使用 `uv pip sync` 会让当前环境严格匹配清单文件，自动安装缺失的包并移除多余的包
```shell
uv pip sync requirements.txt
```


## 现代化项目管理

1. 初始化项目 `uv init`
2. 添加依赖(会自动写入 pyproject.toml) `uv add numpy pandas`
3. 锁定依赖版本 `uv lock`
4. 同步环境 `uv sync`

## 项目迁移最佳实践

```shell
# 1. 创建新环境
uv venv 

# 2. 激活环境
source .venv/bin/activate

# 3. 严格同步依赖
uv pip sync requirements.txt
```
