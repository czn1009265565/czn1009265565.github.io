# Vue3 组件

每个 Vue 组件都是一个独立的 Vue 实例，
具有自己的模板、数据、方法和生命周期钩子，使得组件可以自包含地定义和管理自己的功能和样式。
组件可以扩展 HTML 元素，封装可重用的代码，支持将用户界面拆分成独立和可复用的部分。

特性与优势

- 复用性: 组件可以在不同的地方多次使用，提高代码的复用性和可维护性。
- 封装性: 每个组件都是独立的作用域，可以封装自己的状态和逻辑，避免全局污染。
- 组合性: 可以通过组合多个小组件构建复杂的界面。
- 响应式: 组件的数据响应式地绑定到视图，数据更新时自动更新视图。
- 模块化: 支持模块化开发，可以使用现代前端工具链进行构建和管理。

## 组件注册

### 单 HTML 文件实现
```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>Vue 3</title>
    <!-- 1. 引入 Vue 3 全局版本 -->
    <script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
</head>
<body>
<!-- 2. 根挂载点 -->
<div id="app"></div>
<script>
    const { createApp, ref } = Vue;

    // --- 定义子组件 (WelcomeCard) ---
    // 在没有 .vue 文件时，组件就是一个包含 template, setup 等属性的对象
    const WelcomeCard = {
        template: `
                <div class="card">
                    <h2>{{ title }}</h2>
                    <p>{{ description }}</p>
                </div>
            `,
        setup() {
            // 子组件内部的数据
            const title = ref('欢迎来到 Vue 3');
            const description = ref('这是通过 CDN 引入的单文件模式实现的子组件。');

            return { title, description };
        }
    };

    // --- 定义根应用 (App) ---
    const App = {
        // 父组件的模板，使用子组件标签 <welcome-card />
        template: `
                <div class="app-container">
                    <h1>父组件区域</h1>
                    <!-- 使用子组件 -->
                    <welcome-card />
                </div>
            `,
        // 注册子组件
        components: {
            'welcome-card': WelcomeCard
        },
        setup() {
            console.log('父组件已加载');
            return {};
        }
    };
    // 3. 创建并挂载应用
    createApp(App).mount('#app');
</script>
</body>
</html>
```

## 组件通信

### 单 HTML 文件实现

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>Vue 3 组件通信示例</title>
    <script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
</head>
<body>

<div id="app"></div>

<script>
    const { createApp, ref } = Vue;

    // --- 1. 定义子组件 (WelcomeCard) ---
    const WelcomeCard = {
        // 模板中使用 props 数据，并绑定点击事件
        template: `
                <div class="card">
                    <h2>你好, {{ userName }}!</h2>
                    <p>这是一个通过 Props 接收数据的子组件。</p>
                    <button @click="notifyParent">点击修改父组件数据</button>
                </div>
            `,
        // 声明接收的 Props
        props: {
            userName: {
                type: String,
                required: true
            }
        },
        // 声明可能触发的事件
        emits: ['update-name'],
        setup(props, { emit }) {

            function notifyParent() {
                // 触发事件，传递新名字给父组件
                emit('update-name', 'Vue Master');
            }

            return {
                notifyParent
            };
        }
    };

    // --- 2. 定义根应用 (App) ---
    const App = {
        template: `
          <div class="app-container">
            <h1>父组件区域</h1>
            <p>当前父组件状态: {{ userName }}</p>

            <!--
              1. :user-name="userName" -> 父传子 (Props)
              2. @update-name="handleChangeName" -> 子传父 (Events)
            -->
            <welcome-card
                :user-name="userName"
                @update-name="handleChangeName"
            />
          </div>
        `,
        components: {
            'welcome-card': WelcomeCard
        },
        setup() {
            const userName = ref('Alice');

            // 处理子组件触发的事件
            function handleChangeName(newName) {
                userName.value = newName;
                alert(`父组件收到消息，名字已改为: ${newName}`);
            }

            return {
                userName,
                handleChangeName
            };
        }
    };

    createApp(App).mount('#app');
</script>
</body>
</html>
```

父传子:
- 原理: 父组件通过属性绑定 (:prop-name="data") 将数据传递给子组件。
- 单向数据流: 子组件只能读 Props，不能直接修改。如果子组件尝试修改 props.userName，Vue 会在控制台发出警告。
- 命名规范:
    - JS/Template 中通常使用驼峰命名 (userName)。
    - HTML 属性不区分大小写，所以 DOM 模板中通常使用短横线 (user-name)。Vue 会自动处理这种转换。

子传父:
- 原理: 子组件通过 emit('event-name', payload) 触发一个自定义事件。
- 监听: 父组件通过 @event-name="handler" 监听该事件，并在 handler 函数中接收 payload 数据。
- 解耦: 子组件不需要知道父组件是谁，只需要知道“我发出了一个信号”，这降低了组件间的耦合度

### 工程化 Vue 实现

子组件

```vue
<template>
  <div class="card">
    <!-- 直接使用 props 中的变量 -->
    <h2>你好, {{ userName }}!</h2>
    <p>这是一个通过 Props 接收数据的子组件。</p>
    
    <!-- 点击触发 emit -->
    <button @click="notifyParent">点击修改父组件数据</button>
  </div>
</template>

<script setup>
import { defineProps, defineEmits } from 'vue';

// 1. 定义 Props (父传子)
const props = defineProps({
  userName: {
    type: String,
    required: true
  }
});

// 2. 定义 Emits (子传父)
const emit = defineEmits(['update-name']);

function notifyParent() {
  // 触发事件
  emit('update-name', 'Vue Master');
}
</script>
```

父组件
```vue
<template>
  <div class="app-container">
    <h1>父组件区域</h1>
    <p>当前父组件状态: {{ userName }}</p>

    <!-- 
      简写方式：
      :user-name 是 v-bind:user-name 的简写
      @update-name 是 v-on:update-name 的简写
    -->
    <WelcomeCard 
      :user-name="userName" 
      @update-name="handleChangeName" 
    />
  </div>
</template>

<script setup>
import { ref } from 'vue';
import WelcomeCard from './components/WelcomeCard.vue';

// 父组件的状态
const userName = ref('Alice');

// 处理子组件发来的事件
function handleChangeName(newName) {
  userName.value = newName;
  console.log('父组件更新成功:', newName);
}
</script>
```
